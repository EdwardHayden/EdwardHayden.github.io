Ruinsanity

2024

Background info

Ruinsanity was my second year university project. I worked in a group of six, two programmers (myself included), and four artists. We spent 9 months developing the game with the first four months dedicated to ideation, prototyping, and pre-production, and the other five spent on production and polish. We decided early on to go with the Unreal Game Engine, as we didn't have much experience working in it and all thought it would be a good way of learning.

About the game

Ruinsanity is a singleplayer 3D puzzle exploration game set in a mystical yet ominous temple, with a rich history, cryptic puzzles, and a time bending amulet, which eats away at your sanity. Ruinsanity features a texture changing mechanic that allows the player to see a different texture for the 3D environment in a small area around them. We achieved this effect by using a mix of Unreal's Blueprint system and Unreal's Shader Graph system.

Skills Gained

As this was my first time working with Unreal, some time was spent learning the new engine. However, due to having prior knowledge of Unity, the process was made a lot easier. Unreal's Blueprint system was very intuitive to learn with its block coding and was surprisingly easy to program more complicated mechanics.
